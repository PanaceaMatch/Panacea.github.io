Vue项目结构

Vue CLI 创建的 Vue 项目结构，以及与 Vite + TypeScript 项目的主要差异

一、不同构建工具的项目结构

  1. Vue CLI 创建的 Vue 项目结构（以 Vue 3 为例）

  2. Vite + Vue + TypeScript 项目 的项目结构

二、核心不同点

目录/文件 Vue CLI 项目 Vite + TS 项目 核心差异说明
静态资源目录 分 public/ 和 src/assets/ 只有 public/ Vue CLI 中：
- public/：不经过 webpack 处理，直接复制到 dist。
- src/assets/：会被 webpack 处理（如图片 base64、CSS 打包）。

Vite 中：
- 只有 public/ 用于静态资源（如 favicon.ico）。
- src/ 下的资源（图片、CSS）直接通过 ES 模块引用（无需区分 assets），Vite 自动处理。
入口文件位置 public/index.html（模板）+ src/main.js index.html（根目录）+ src/main.ts Vue CLI 中：
- index.html 是 webpack 模板，打包时注入 bundle。
- 入口 JS 是 main.js。

Vite 中：
- index.html 是原生 ES 模块入口（脚本标签加 type="module"），直接被浏览器加载。
- 入口是 main.ts（TS 原生支持）。
配置文件 vue.config.js + babel.config.js vite.config.ts Vue CLI 中：
- vue.config.js：封装 Webpack 配置（如 devServer、代理）。
- babel.config.js：ES6+ 转 ES5（需手动配置）。

Vite 中：
- vite.config.ts：直接配置 Rollup（生产）和 ESBuild（开发），语法简洁。
- 无需 babel.config.js（ESBuild 自动转译主流浏览器）。
TypeScript 相关文件 需手动创建 shims-vue.d.ts 自动生成 vite-env.d.ts Vue CLI + TS 中：
- 需手动创建 shims-vue.d.ts 声明 Vue 组件类型（否则 TS 报错）。

Vite + TS 中：
- 自动生成 vite-env.d.ts，声明 Vite 环境变量（如 import.meta.env）和 Vue 组件类型，无需手动配置。
状态管理目录 src/store/（默认 Vuex） 需手动创建 src/stores/（推荐 Pinia） Vue CLI 中：
- 新建项目勾选 Vuex 后，自动生成 src/store/index.js。

Vite 中：
- 不默认集成状态管理，需手动安装 Pinia 并创建 src/stores/ 目录。

三、总结

对比维度 Vue CLI 项目 Vite + TS 项目
项目结构 目录划分细（public/assets 分离），配置文件多（vue.config.js、babel.config.js） 目录简洁（仅public/静态资源），配置文件少（仅vite.config.ts）
使用体验 启动慢、热更新中等、配置繁琐、TS 支持需额外配置 启动极快、热更新即时、配置简洁、TS 原生支持
核心优势 生态成熟、兼容 Vue 2/3、Webpack 配置灵活（适合复杂定制） 开发效率高、构建速度快、原生 TS 支持（适合 Vue 3 新项目）
适用场景 维护旧项目（Vue 2）、需要深度定制 Webpack 配置的项目 新建 Vue 3 项目、追求高效开发体验的项目
